# Корпоративная База Сотрудников

Современное SPA-приложение на **Vue 3 (Composition API)** для отображения и фильтрации списка сотрудников. Данные грузятся из реальной базы **Supabase (PostgreSQL)**.

## ⚙️ Жизненный цикл (Как это работает)

### 1. Инициализация и Загрузка (Data Fetching)
* При открытии страницы монтируется контроллер `UsersView.vue`.
* Срабатывает хук `onMounted`, который вызывает экшен стора `store.fetchUsers()`.
* **Store** включает загрузочные скелетоны (Skeleton Loading) и обращается к **Service Layer** (`userService.ts`).
* `userService` использует клиент `@supabase/supabase-js` для HHTP-запроса к таблице `users` в СУБД PostgreSQL.
* Полученный массив оседает в реактивном `state` стора (`users`).

### 2. Реактивное кеширование (Core Logic)
В `useUsersStore.ts` живет главный движок — `computed` свойство `filteredUsers`.
Оно автоматически пересчитывается только тогда, когда меняются его зависимости — базовый массив `users`, текстовый запрос `searchQuery` или критерии `filter`. На оригинальный массив мутации не влияют, что оптимизирует память и производительность (O(N)).

### 3. Пользовательское взаимодействие (Composables)
Логика ввода отделена от UI компонентов в **Composables**:
* **`useSearch.ts`**: Обрабатывает ввод в строку поиска. Главная фича — **Debounce (350ms)**. Запрос в `store` отправляется не на каждую напечатанную букву, а только после того как пользователь перестал печатать. 
* **`useFilter.ts`**: Обрабатывает выбор города или компании, извлекая доступные уникальные значения (через `new Set()`) напрямик из стора.

### 4. Рендер UI (View Layer)
Компонент `UserList.vue` получает массив через `Props` от Оркестратора и, используя алгоритм Virtual DOM (с обязательным `:key="user.id"`), атомарно обновляет и анимирует исключительно те карточки, которые прошли фильтрацию.

## 🚀 Стек Технологий

* **Vue 3** (Composition API, `<script setup>`)
* **TypeScript** (Строгая типизация контрактов и компонентов)
* **Pinia** (State Management без мутаций)
* **Supabase** (BaaS, PostgreSQL, SDK клиент)
* **CSS3 / Glassmorphism** (Адаптивный Grid, backdrop-filter)
* **Vite** (Быстрый HMR бандлер)

## 🏁 Запуск проекта

Убедитесь, что в корне проекта (рядом с `package.json`) есть файл `.env` с ключами `VITE_SUPABASE_URL` и `VITE_SUPABASE_ANON_KEY`.

```bash
npm install
npm run dev
```
